3.1 Review the benefits and drawbacks of IT tools and systems used in terms of productivity and efficiency

The tools used were as following:

  • Visual Studio Code: It is a light and a very versatile editors. The extensions available are really handy. With support for hundreds of languages, VS Code helps you be instantly productive with syntax highlighting, bracket-matching, auto-indentation, box-selection, snippets, and more.
  • Codepen: CodePen is a social development environment. At its heart, it allows you to write code in the browser, and see the results of it as you build. A useful and liberating online code editor for developers of any skill, and particularly empowering for people learning to code.
  • Trello: It is an excellent way of implementing Kanban and even working on the project text based RPG it helped keeping track of the progress and issues. You can assign a particular task to someone as well which reminds them they have the responsibility.

3.2 Describe ways to improve productivity and efficiency

There are plenty of ways to increase productivity when working on individual projects. Some are mentioned as following:

  • Track and limit how much time you're spending on tasks.
  • Take regular breaks.
  • Set self-imposed deadlines.
  • Quit multitasking.
  • Give up on the illusion of perfection.
One other important way is to not treat your first release as your final realse, get a minimum viable product running as soon as possible and then add features to it.

3.4 Describe how you would go about testing digital solutions

To test this digital solution I would open the page in different browsers and see how it responds to them as support for the browsers is different. The other thing is opening all the possible links so see they are working in both browsers. Since the page is pretty much static so the issues that could arise are minimal. Since also the page doesn't take any input we don't need to worry about things like SQL injection but generally whenever testing a webpage or a digital solution the check list is as following:

  • Functionality Testing
  • Usability testing
  • Interface testing
  • Compatibility testing
  • Performance testing
  • Security testing
